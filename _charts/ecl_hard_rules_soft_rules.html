Framework 02
Hard rules vs soft rules
Your pipeline must survive soft rules being wrong
Hard rules
System enforces
PRIMARY KEY
UNIQUE INDEX
FOREIGN KEY
NOT NULL
CHECK
WATERLINE — query information_schema to find it
Soft rules
Someone told you it's true
"Quantities are always positive"
"Status goes pending → confirmed → shipped"
"Only open invoices get deleted"
"updated_at is always set"
"Emails are unique"
"A line's status matches the header"
When a soft rule breaks
Load the data. Surface the violation. Don't fix it in the pipeline.
