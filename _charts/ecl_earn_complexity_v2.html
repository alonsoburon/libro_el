Framework 05
Earn complexity
Start simple. Add complexity only when forced.
1
Full replace every run
Stateless, idempotent, pure. No cursor, no state, no drift.
DEFAULT
Full table too big for the schedule window
2
Scoped full replace
Replace a date range instead of the whole table.
Scope boundary is metadata-driven, not calendar-driven
3
Rolling window replace
Replace based on _extracted_at or activity signal.
Table genuinely too large to full-replace at required cadence
4
Cursor-based incremental
Track what changed since last run. Requires reliable cursor. Accumulates purity debt.
Hard deletes exist at source
5
Cursor + delete detection + periodic full reset
Maximum complexity. The upper bound of what's reasonable.
CEILING
Principle
Each step is triggered by a concrete "no" in the decision tree. If you can't name the trigger, you haven't earned the complexity.
