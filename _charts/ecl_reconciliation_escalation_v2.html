Framework 11
Reconciliation escalation
Start cheap. Escalate only when the cheaper level signals a problem.
1
Row count
CHEAP
COUNT(*) source vs destination. Catches missing rows, duplicates, dropped partitions. Misses value drift.
↓ count mismatch triggers aggregate
2
Aggregate checks
MEDIUM
SUM(amount), MAX(updated_at), COUNT(DISTINCT id). Catches value drift row count misses.
↓ aggregate mismatch triggers hash
3
Hash comparison
EXPENSIVE
Hash every row, compare. Catches any difference at any granularity. Nuclear option.
Escalation logic
Count mismatch → aggregate check. Aggregate mismatch → hash or full ID comparison. Reserve level 3 for critical tables or periodic audits.
